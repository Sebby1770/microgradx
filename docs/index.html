So… what does this actually do?

Imagine you got a robot puppy for your birthday. 🐕

The puppy doesn't know any tricks yet. You want to teach it to sit, or fetch the ball, or tell the difference between a cat and a dog in a photo.

Here's how robots learn:

1
You show the puppy a picture and ask: "is this a cat?"
2
The puppy guesses. At first, it guesses wrong. A lot.
3
You tell it: "no — that's a dog."
4
The puppy adjusts its tiny robot brain a little bit, so it'll be slightly less wrong next time.
5
Show it a million pictures. After a while… the puppy is amazing at telling cats from dogs.

The really tricky part is step 4 — figuring out exactly which parts of the puppy's robot brain need to change, and by how much. That takes a lot of math (the kind grown-ups call "calculus").

MicroGradX is the calculator that does that math. It's a tiny tool that lets people who want to teach robots focus on the fun part — designing the robot's brain — and not the boring math part.

✨ Big famous tools that do the same thing: PyTorch (Meta), TensorFlow (Google), JAX (also Google). They're millions of lines of code. MicroGradX does the same core thing in 3,400 lines you can actually read in an afternoon.

What's in the box

Real building blocks. No toys.

Everything you'd reach for in PyTorch — implemented from scratch with the math right there in the source.

⚡

Dynamic autograd

Define-by-run computation graph with broadcasting. Iterative reverse-topological backward.

🧱

Real layers

Linear, Conv2d (im2col), MaxPool2d, LayerNorm, RMSNorm, Dropout, Embedding, MultiHeadAttention.

📈

Optimizers

SGD with Nesterov momentum, AdamW with bias correction, Lion. Plus L2 / L∞ gradient clipping.

🎯

Gradient checking

Built-in fp64 numerical-vs-analytical gradient checker. Every primitive is tested with it.

🚂

Training utilities

Trainer with gradient accumulation, mixed-precision plumbing, DataLoader + augmentation transforms.

📦

ONNX export

Trace your model and emit a real ONNX graph (or JSON if onnx isn't installed). Works for the documented op set.

Looks familiar

If you've used PyTorch, you already know how to use this.

Same vocabulary. Same patterns. Subclass nn.Module, write a forward, run loss.backward(), call opt.step().

The difference: when something feels weird, you can open the source and the answer will be in there — readable, commented, with the math derivation right next to the code.

mnist_mlp.py 
import microgradx as mg
from microgradx import nn, optim

# 1.  define a model
model = nn.Sequential(
    nn.Linear(784, 128),
    nn.ReLU(),
    nn.Linear(128, 10),
)

# 2.  pick an optimizer
opt = optim.AdamW(model.parameters(), lr=1e-3)

# 3.  train
for x, y in loader:
    logits = model(mg.Tensor(x))
    loss   = nn.cross_entropy(logits, y)

    model.zero_grad()
    loss.backward()
    opt.step()
The whole map

Every file has a job. Here's what's where.

No mystery folders, no generated code, no plugin system. The whole framework is laid out flat.

microgradx/ ├── tensor.py # Tensor + autograd driver ├── backend/ # numpy ↔ cupy switch ├── autograd/ │ ├── function.py # base Function/Context │ ├── ops.py # every primitive forward + backward │ └── grad_check.py # fp64 numerical gradient checker ├── nn/ │ ├── module.py # Module, Sequential, ModuleList │ ├── linear.py # Linear │ ├── conv.py # Conv2d, MaxPool2d (im2col impl) │ ├── norm.py # LayerNorm, RMSNorm │ ├── dropout.py # Dropout │ ├── activation.py # ReLU, GELU, Sigmoid, Tanh, Softmax │ ├── embedding.py # Embedding lookup │ ├── attention.py # MultiHeadAttention + scaled dot-product │ └── loss.py # CrossEntropy (fused), MSE ├── optim/ │ ├── sgd.py # SGD (+momentum, +Nesterov, +weight decay) │ ├── adamw.py # AdamW │ └── lion.py # Lion ├── data/ # Dataset, DataLoader, transforms ├── training/ # Trainer, AMP, GradScaler └── export/ # ONNX exporter
